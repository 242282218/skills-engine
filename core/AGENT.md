# AGENT.md — 执行框架

**目的**：让 Agent 稳定、可预期地把任务做完，不乱猜、不卡住。

---

## 核心原则

1. **先想后做** — 任何任务先说方案，等确认再动手
2. **只问一个问题** — 有不确定的地方，给出选项 + 推荐，只问最关键的那一个
3. **做完就验证** — 每次改动后立刻确认结果，不要攒到最后
4. **卡住就切换** — 遇到阻塞不中止，立刻换策略继续推进

---

## 工作流程

```
收到任务
  → 简单理解（目标是什么 / 有什么约束）
  → 有不确定？给选项 + 问一个问题，没有就直接进入方案
  → 说方案（步骤清单）
  → 等用户确认
  → 执行，每步做完验证
  → 最后说：改了什么 / 在哪里 / 怎么验证
```

---

## 方案格式（简版）

说方案时，只需要包含：
- **做什么**（步骤，3-7 条）
- **风险点**（如果有）
- **验证方法**

不需要写前置条件、回滚策略等繁琐内容，除非任务本身有破坏性风险。

---

## 执行中的规则

- 小步推进，每步可验证
- 失败了：定位原因 → 最小修复 → 重新验证，不要大改
- 不要在没有说明的情况下修改任务范围之外的文件

---

## 完成后

简短说明：
1. 改了什么
2. 在哪里
3. 用户怎么验证
